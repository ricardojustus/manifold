# The operator owns criticality, security-machinery depth, and operating posture — ask, don't assume

**Rule.** Some decisions are the operator's to make; the agent may not quietly make them by building as if they were settled. This is an AUTHORITY boundary beside the YAGNI machinery (`right-sized-engineering` asks "is this proportionate?"; this asks "whose call is it?").

**Reserved to the operator**: whether a component is mission-critical, how much downtime it tolerates, whether it can be turned off; how much security/assurance machinery to build and what residual risk to accept; the operating posture generally — any choice that materially changes the solution class, delays a capability, adds recurring cost/quota, or stands up a new guard / recovery / migration / process / security subsystem. Routine implementation complexity INSIDE an already-chosen posture stays the agent's (retry counts, timeouts, validation, permission checks). **The agent may NOT infer a reserved call from a risk label, architecture, industry practice, reviewer/council consensus, or prior artifact approval** — a label is not information.

**Security invariants — binding by default, lowered only by an informed exception.** An invariant whose one violation is irreversible (a no-exfiltration posture; the enforcement invariants) remains binding absent an explicit, informed operator exception. Never silently weakened; while a decision pends, neither open new exposure nor lower an existing control; the standard control is built by default — only gold-plating beyond it routes to the operator. The operator may accept a posture below such an invariant ONLY via a **finite, totally-informed exception**: a bounded disclosure they explicitly accept, naming what could leak/fail · by what path · to which destination class · why unrecoverable · the simplest invariant-preserving option and its real cost · mitigations + remaining risk · every material known warning · what remains unknown. Structural-ratification tier (recorded packet + brief, never an inline "fine"); cannot erase an obligation outside the operator's authority (third-party/contractual constraints are disclosed, not overridden).

**Present, don't assume — in the operator's terms.** Surface the reserved call with its consequence concrete (what breaks, to what, how badly, recovery in operator labor and downtime, what the machinery buys) — never a bare label. Always find and present the simplest way beside any heavier option. Cheap technique: the frame-reset (`spec-writing` Step 0) — restate the change stripped of solution nouns and risk labels; the machinery must justify itself from scratch. Don't stand up a new review invocation per decision — the operator, not another model, holds the missing risk tolerance.

**The teeth — the plan-lock gate.** A plan cannot lock while operator-visible machinery justified by a criticality / recovery / security-depth posture lacks a cited **operator POSTURE RECEIPT**: a stable pointer to the complete decision packet AND the operator's explicit response, its failure/recovery story grounded in current-state docs, naming the simplest alternative the posture was chosen against. NOT a receipt: a risk tag, a generic GO, industry practice, consensus, approval of unrelated rulings. Missing receipt = **"operator posture unresolved"** — never a conservative default that builds the safest machinery unasked. The classification stays factually visible; only its MACHINERY CONSEQUENCES wait — a recommendation routed to the operator, never a license to silently skip an owed hardening; the gate fires on the consequence, so machinery under a neutral name ("robustness") is caught the same way. Backstop: `spec-writing` Step 0 for machinery appearing in a spec with no plan behind it.

**Bounding**: trigger on operator-visible consequence, not keywords. A recorded ruling counts as "asked" for its exact scope; a class-wide ruling is a standing receipt for its class; re-ask only when the recovery story changes or a materially different protection class is added. A receipt whose story is ungrounded, or whose cheap alternative was never named, does not count — even with a GO.

Pairs with: `right-sized-engineering` (simplicity kernel + classification check) · `operator-translation` (packet completeness; criticality and security-depth are the facts most often withheld) · `ask-vs-decide` (reserved-authority calls are a halt trigger, never decide-and-park).
